Question 3 (22 marks)

This question assesses your understanding of Unit 2, in particular use of abstract data types (ADTs) and solving computational problems using Python implementations.

For the parts of this question that involve writing Python code, we have provided two Python files (available from the Supporting files for TMAs [Tip: hold Ctrl and click a link to open it in a new tab. (Hide tip)]   page of the M269 website). Python file M269_14J_TMA01_Q3_Grid.py contains a class that you will use, but you will not need to change this file. Copy the other Python file M269_14J_TMA01_Q3.py and rename it M269_14J_TMA01_Q3_OUCU.py, where ‘OUCU’ is your OU computer username, e.g. abc123. Keep both Python files in the same folder.

Make sure that you add your answers for the question parts at the places indicated in the file M269_14J_TMA01_Q3_OUCU.py. When you have completed all the parts of this question, include this file, together with the supplied file M269_14J_TMA01_Q3_Grid.py, in your TMA zip file. Also, copy into your Solution Document the code for the functions we ask you to write.

For this question we want you to develop some Python functions that could be used in a simplified version of the game for two players known as Battleships. This has been played for many years under various names in a variety of countries, originally on paper grids. For more information about the standard game and its history see http://entertainment.howstuffworks.com/ leisure/ brain-games/ battleship-game3.htm or http://en.wikipedia.org/ wiki/ Battleship_(game).

For our purposes we will define a simplified form of the game, but if you read the references about the standard game you should be able to see how our game could be extended to a more complex and realistic game.

Our simplified game works like this. Each player has a grid of the same size, with each grid cell initially empty. Each player adds the letter ‘S’ in a number of cells on their grid to represent the positions of a number of ‘ships’ (see Figure 1).

Figure 1 Example of initial grid for Battleships game
Long description

Each player tries to guess where the other player’s ships are and then specifies a grid cell for ‘attack’. If this grid cell contains a ship then the opponent has to acknowledge this by responding with ‘hit’ and if the cell is empty they respond with ‘miss’. Players take turns to attack, specifying one cell each time. If a cell containing a ship has been hit, the ‘owner’ of that grid marks the cell to record that the ship has been sunk (replacing the original letter ‘S’ with an ‘X’).

As a basic component for the game, we are going to use an n by n grid. Following good practice, we have implemented the Grid ADT in Python as a class. We are using encapsulation to hide the internal representation of the grid from the user or client. You should have read about the advantages of this approach in Unit 2. The Python file M269_TMA01_14J_Q3_Grid.py, supplied for this question, contains the code for a class Grid that implements this ADT. You do not have to understand the code for this class in detail for the purposes of this TMA.

This Grid class contains a constructor __init__(n) for creating a new n × n grid (i.e. it has n rows and n columns, with rows and columns numbered from 1 to n).

It also contains methods as follows:

  • setCell(r, c, aValue) sets the cell at row r, column c to aValue
  • getCell(r, c) returns the value of the cell at row r, column c
  • getSize() returns the size of the (square) grid – i.e returns n for an n × n grid
  • __str__() (which you should be familiar with after studying Unit 1) returns a string that can be printed to give a simple visual representation of the state of the grid.

An empty 4 by 4 grid can be created using the following Python code:

myGrid = Grid(4)

We can then show the state of the grid with the following code:

print(myGrid)

which will print the following to the command output:

-----------------

|   |   |   |   |

-----------------

|   |   |   |   |

-----------------

|   |   |   |   |

-----------------

|   |   |   |   |

-----------------

To add an ‘S’ representing a ship to the grid at row 1, column 3, we can write this code:

myGrid.setCell(1, 3, 'S')

We can then show the new state of the grid with the following code:

print(myGrid)

which will print the following to the command output:

-----------------

|   |   | S |   |

-----------------

|   |   |   |   |

-----------------

|   |   |   |   |

-----------------

|   |   |   |   |

-----------------

  • b. 

    • i.Suppose you have a sequence of locations of ships, with each location given as a pair of row and column positions. You need to write a function that creates an empty n × n grid and then initialises it with a ship at each of the specified locations, with a ship being represented by an 'S' in the specified cell. In other words, you need to solve the following computational problem:

      Name: MakeShipsGrid

      Inputs: An integer n and a sequence L of ship locations, where each location is a tuple (r, c) such that each of r and c is an integer between 1 and n inclusive.

      Outputs: An n × n grid such that cells in locations specified by the sequence L contain an 'S' and all other cells are empty (denoted by ' ').

      For example, if the inputs are 3 and [(1,2), (2,1), (3,1), (3,3)], the output should correspond to the 3 by 3 grid shown in Figure 2.

Given a particular grid cell specified by its row and column, the function should return 'Hit' or 'Miss' depending on whether or not there is a ship (denoted by 'S') at that cell. If there is a ship at the specified cell then the 'S' should be replaced by an 'X' to denote a sunken ship. If there was already a sunken ship at the specified cell this should be treated in the same way as a ‘Miss’. In other words, you need to solve the following computational problem:

      Name: Attack

      Outputs: The string 'Hit' if there was a ship in the grid at position (r, c) and the string 'Miss' if not. If there was a ship at position (r, c) then the value stored at that position in the grid will have been changed to show the ship has sunk (see Figure 3 for an example of this).
